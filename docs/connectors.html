Connectors & Ingestion

Ingest documents from enterprise sources into the system of record and system of recall.

Overview

Engram supports ingestion from multiple enterprise sources via connectors that integrate with Unstructured.io and write to both the:

  • System of Record: Azure Blob Storage (raw artifacts + metadata)
  • System of Recall: Zep (chunks as facts for retrieval)

Connector Registration Flow

  1. Navigate to Ingestion

    Go to "Ingestion > Connectors" in the Navigation UI.

  2. Select Source Type

    Choose SharePoint, S3, Google Drive, etc.

  3. Configure Credentials

    OAuth redirect â†’ refresh token stored in Key Vault. API keys stored as connector-{id}-credentials.

  4. Set Scope

    Specify folders, prefixes, sites to ingest.

  5. Configure Schedule

    On-demand, daily, or hourly sync.

  6. Save Connector

    API stores config + credentials in Key Vault.

API Endpoints

# Register new connector
POST /api/v1/connectors
{
  "type": "sharepoint",
  "name": "Engineering Docs",
  "credentials": {...},  # Stored in Key Vault
  "scope": {"site": "...", "library": "..."},
  "schedule": "daily"
}

# Get connector sync status
GET /api/v1/connectors/{id}/status

# Trigger manual sync
POST /api/v1/connectors/{id}/sync

# List ingestion jobs
GET /api/v1/ingestion/jobs

ACL Propagation

Ingestion retains access rules from source systems:

{
  "source": "sharepoint",
  "filename": "Q4-Planning.docx",
  "metadata": {
    "source_acl": ["engineering@company.com:read", "pm@company.com:read"],
    "tenant_id": "company-tenant",
    "domain": "engineering"
  }
}
  • Source ACL â†’ Blob metadata: Preserved for audit
  • Source ACL â†’ Zep fact metadata: For retrieval filtering
  • Tenant mapping: Map source users/groups to Engram tenants/roles
